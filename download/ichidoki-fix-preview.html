Why this happens

  1. Your /api/auto-import returns a URL like https://archive.org/download/.../file.mp4.
  2. archive.org/download returns a 302 redirect to a CDN host like ia801906.us.archive.org.
  3. The redirect response includes Access-Control-Allow-Origin: *, but the CDN host does not.
  4. When crossOrigin="anonymous" is set on a <video> element, the browser requires every response in the redirect chain to send valid CORS headers — including the final CDN response.
  5. Because the CDN response is missing Access-Control-Allow-Origin, the browser blocks the media, the error event fires, and the element ends up in the disabled Unable to play media. state.
  6. Removing crossOrigin makes the request a plain no-cors media fetch, which browsers always allow for <video> playback regardless of response headers.

Same source URL, same browser, same network — only the crossOrigin attribute differs. Full write-up: ichidoki-episode-playback-fix.md
